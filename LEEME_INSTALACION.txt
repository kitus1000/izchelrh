# GUÍA DEFINITIVA DE INSTALACIÓN Y DESPLIEGUE - SISTEMA EL EXPEDIENTE

Esta guía explica paso a paso desde cero cómo levantar todo el ecosistema (Base de datos, Web y App Móvil) en una nueva computadora o cuenta.

---

## 1. PASOS EN SUPABASE (Base de Datos)
Para levantar la base de datos desde cero en un proyecto nuevo, entra a la carpeta **/sql** y ejecuta los archivos en el Editor SQL de Supabase siguiendo estrictamente el orden de los números:

1. **01_ESQUEMA_BASE.sql**: Crea las tablas principales, catálogos iniciales y el sistema de perfiles.
2. **02_MODULO_CHECADOR.sql**: Extiende la base de datos para habilitar el sistema de tablets y QR.
3. **03_ADSCRIBCION_EXT.sql**: Agrega el control de jefaturas.
4. **04_LIMITE_FALTAS.sql**: Configura la tolerancia para faltas automáticas.
5. **05_TIPOS_REGRESO.sql**: Registra los trámites de retorno (Permisos).
6. **06_TOLERANCIA_RETORNOS.sql**: Ajusta los tiempos de gracia para registros.
7. **07_POLITICAS_SEGURIDAD.sql**: Activa los permisos finales (RLS) para que las Apps puedan leer y escribir datos.
8. **08_SEMILLAS_CONFIG.sql**: Inserta los tipos de incidencias (Vacaciones, Faltas, Incapacidades) y solicitudes necesarias.
9. **09_DIAS_FESTIVOS_2026.sql**: Configura el calendario de días feriados oficiales de México para este año.
10. **10_REPARAR_LLAVES_FORANEAS.sql**: Script de mantenimiento para asegurar que todas las relaciones entre empleados y tablas funcionen correctamente.

*Nota: Es fundamental ejecutar los 10 archivos en orden para que el sistema funcione al 100%.*

## 2. PASOS EN LA PC - APLICACIÓN WEB (Carpeta /web)
1. **Instalar dependencias clave:**
   - [Node.js](https://nodejs.org/) (Descarga e instala la versión LTS, e.g., v20 o v18).
   - [Git](https://git-scm.com/) (Para poder subir a GitHub después).
   - [Visual Studio Code](https://code.visualstudio.com/) (Editor de texto).
2. **Abrir el proyecto:**
   - Entra a la carpeta `/web` de este respaldo y haz clic derecho -> "Abrir en la terminal" o abre VS Code y arrastra la carpeta `/web` ahí.
3. **Instalar librerías:**
   - En la terminal de VS Code (presionando `Ctrl + \``), escribe:
     `npm install`
4. **Configurar Entorno (.env.local):**
   - Asegúrate de que exista un archivo llamado `.env.local` en la raíz de `/web`.
   - Su contenido debe ser (reemplaza con tus llaves reales si creaste un Supabase nuevo, o usa las del respaldo):
     ```
     NEXT_PUBLIC_SUPABASE_URL=tu_url_de_supabase
     NEXT_PUBLIC_SUPABASE_ANON_KEY=tu_anon_key_de_supabase
     GEMINI_API_KEY=tu_clave_de_gemini
     GROQ_API_KEY=tu_clave_de_groq
     ```
5. **Probar en la PC:**
   - Ejecuta: `npm run dev`
   - Abre tu navegador en `http://localhost:3000`. ¡El sistema debería cargar!

---

## 3. PASOS PARA DESPLEGAR LA WEB EN VERCEL (Internet)
Para que cualquier persona pueda entrar a la web desde su celular o PC:
1. **Subir código a GitHub:**
   - Entra a [GitHub](https://github.com), crea un nuevo repositorio (ej. `rh-system-prod`).
   - Sube todo el contenido de la carpeta `/web` a ese repositorio.
2. **Conectar Vercel:**
   - Entra a [Vercel.com](https://vercel.com) e inicia sesión con tu GitHub.
   - Dale a "Add New Project" e importa el repositorio `rh-system-prod`.
3. **Variables de Entorno (¡CRÍTICO!):**
   - Antes de darle Deploy, abre la sección **Environment Variables**.
   - Agrega exactamente las mismas 4 variables que tienes en tu archivo `.env.local` (NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, GEMINI_API_KEY, GROQ_API_KEY).
4. **Desplegar:**
   - Clic en Deploy. En unos minutos tendrás un enlace (ej. `rh-system.vercel.app`) disponible globalmente.

---

## 4. PASOS PARA LA APP ANDROID CHECADOR (Carpeta /android)
1. **Abrir proyecto:**
   - Abre VS Code y arrastra la carpeta `/android` al editor.
2. **Instalar librerías:**
   - Abre una terminal nueva en VS Code.
   - Ejecuta: `npm install`
3. **Configurar Entorno (.env):**
   - Verifica que en la raíz de `/android` exista un archivo `.env` con las variables apuntando al Supabase correcto:
     ```
     EXPO_PUBLIC_SUPABASE_URL=tu_url_de_supabase
     EXPO_PUBLIC_SUPABASE_ANON_KEY=tu_anon_key_de_supabase
     ```
4. **Correr la App en el Celular (Desarrollo):**
   - En la terminal ejecuta: `npx expo start`
   - Aparecerá un código QR grande.
   - Descarga la app **"Expo Go"** en la Play Store de un celular Android.
   - Abre la app, escanea el QR y verás tu Checador corriendo. Siempre que tu PC y tu Celular estén en el mismo WiFi.
5. **Generar el APK Final para instalar directo sin Expo Go:**
   - Necesitas tener una cuenta en [Expo (EAS)](https://expo.dev).
   - En la terminal corre `npm install -g eas-cli`
   - Inicia sesión con `eas login`
   - Genera el archivo instalable corriendo: `eas build -p android --profile preview`
   - Al terminar, te dará un enlace para descargar el archivo `.apk` que podrás enviar e instalar permanentemente en las tablets o celulares de la empresa.

---
Con estos pasos tienes el control absoluto desde la base de datos hasta las instalaciones físicas en la empresa.
